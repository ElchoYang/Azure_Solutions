# Azure Files vs Azure Blob Storage 费用对比分析

> **文档说明**：本文档基于 Microsoft Azure 官方定价（East US 区域，2025-2026 参考价格），以美元计价，区域和汇率不同可能存在差异。
> 建议最终报价以 [Azure 定价计算器](https://azure.microsoft.com/zh-cn/pricing/calculator/) 为准。

---

## 一、服务定位对比

| 对比维度 | Azure Files | Azure Blob Storage |
|---------|------------|-------------------|
| **服务类型** | 托管文件共享（SMB/NFS/REST） | 对象存储（Block/Append/Page Blob） |
| **访问协议** | SMB 3.x、NFS 3.0、REST API | REST API、HTTPS |
| **典型用途** | 企业文件共享、应用迁移 Lift & Shift、容器持久化存储 | 图片/视频/文档存储、备份归档、日志收集、静态网站 |
| **目录结构** | 传统文件系统目录树（真实路径） | 扁平对象+虚拟目录（前缀模拟） |
| **挂载方式** | 可直接挂载为网络驱动器（Windows/Linux） | 不支持直接挂载（需工具或代码访问） |
| **最大容量** | 标准：最高 100 TiB；高级：最高 102.4 TiB | 实际无限制（PB 级别） |

## 八、成本差异总结

### 核心结论

| 对比维度 | 结论 |
|---------|------|
| **纯存储成本** | Blob Storage **更低**，归档层可比 Files 便宜 **7–30 倍** |
| **SMB 文件共享溢价** | Azure Files 标准热层比 Blob 热层贵约 **40%**，高级 SSD 贵 **5-7 倍** |
| **操作费用** | 两者相近，高频操作场景 Files 事务优化层略有优势 |
| **数据出站** | 两者相同，$0.087/GB |
| **冗余溢价** | 两者冗余价格结构相同，ZRS ≈ +25%，GRS ≈ +100% |

### 何时选 Azure Files？

- 应用需要 **SMB/NFS 挂载**（Windows 服务器、Linux 应用无改造迁移）
- 需要 **Active Directory + NTFS ACL** 权限控制
- 已有本地文件服务器迁移 Azure（Lift & Shift）
- Kubernetes / AKS 持久化存储

### 何时选 Blob Storage？

- 存储**非结构化数据**（图片、视频、日志、备份）
- 需要**最低存储成本**（冷/归档层）
- 构建**云原生应用**（REST API 直接访问）
- 需要**静态网站**或 **CDN 加速**
- 需要**生命周期自动分层**管理
